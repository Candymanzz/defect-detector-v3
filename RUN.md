# Как запустить это приложение (инструкция «с нуля»)

Этот текст написан так, будто вы **никогда не запускали программы из чёрного окна** и не знаете слов вроде «Maven». Если выполнять шаги **по порядку** и **не пропускать**, оркестратор можно собрать и запустить.

---

## Слова, которые встретятся один раз

- **Папка проекта** — это папка, в которой лежат папки `config`, `orchestrator-java`, `java-geometry-service` и файл `RUN.md`. Запомните, где она у вас на диске. Пример: `D:\VisionMaster4.4.0\megaOrchestratoraue228-main`.
- **Корень репозитория** — то же самое, что «папка проекта».
- **PowerShell** — программа Windows с синим или чёрным окном, куда вводят команды буквами с клавиатуры.

---

## Шаг 0. Убедиться, что вы в правильной папке

1. Откройте проводник Windows.
2. Зайдите в папку проекта (ту, где есть папка `config`).
3. В адресной строке проводника **скопируйте полный путь** (например `D:\...\megaOrchestratoraue228-main`). Он понадобится.

Если папки `config` рядом нет — вы не в том месте. Найдите архив/клон проекта и откройте его корень.

---

## Шаг 1. Установить Java 17 (обязательно)

1. Вам нужна **Java JDK версии 17** (не 8, не 11 — именно 17, иначе сборка может не взлететь).
2. Скачайте установщик JDK 17 с официального сайта Oracle или используйте **Eclipse Temurin 17** (удобный вариант): в поиске браузера наберите `Eclipse Temurin 17 JDK Windows` и скачайте установщик под Windows x64.
3. Установите, следуя мастеру. В установщике, если спросят, **включите** опцию вроде «Add to PATH» / «Set JAVA_HOME variable» — это важно.
4. Закройте все окна PowerShell, если они были открыты, и откройте новое (см. шаг 2), чтобы Windows увидела Java.

**Проверка.** Откройте PowerShell и введите команду (потом нажмите Enter):

```text
java -version
```

В ответе должна быть строка с цифрой **17** (например `openjdk version "17"` или `java version "17"`). Если пишет «команда не найдена» — Java не в PATH; переустановите JDK и снова включите добавление в PATH, либо попросите знакомого настроить переменную среды.

---

## Шаг 2. Установить Maven (нужно, чтобы «собрать» jar-файлы)

1. В браузере найдите страницу загрузки **Apache Maven** (Binary zip для Windows).
2. Распакуйте архив в простую папку, например `C:\tools\apache-maven-3.9.9`.
3. Нужно добавить Maven в PATH. Самый простой путь для новичка:
   - Нажмите Win, наберите `переменные среды`, откройте «Изменение системных переменных среды».
   - Кнопка «Переменные среды».
   - В «Системные переменные» найдите `Path`, нажмите «Изменить», «Создать», вставьте путь к папке `bin` внутри Maven, например `C:\tools\apache-maven-3.9.9\bin`.
   - Сохраните всё, закройте окна.
4. Откройте **новое** окно PowerShell.

**Проверка:**

```text
mvn -version
```

Должна появиться версия Maven и снова упоминание Java 17. Если «mvn не является внутренней или внешней командой» — PATH для Maven не настроен, вернитесь к пункту 3.

---

## Шаг 3. Открыть PowerShell уже внутри папки проекта

1. Откройте проводник и зайдите в **корень проекта** (где лежит папка `config`).
2. Щёлкните **по пустому месту** в адресной строке (где написан путь к папке), чтобы путь выделился.
3. Наберите на клавиатуре слово `powershell` и нажмите Enter.  
   Откроется окно PowerShell, и **текущая папка уже будет папкой проекта**. Это нужно каждый раз перед сборкой и запуском.

Альтернатива: открыть PowerShell где угодно и выполнить (подставьте **свой** путь в кавычки):

```powershell
Set-Location "D:\VisionMaster4.4.0\megaOrchestratoraue228-main"
```

Проверка, что вы на месте:

```powershell
Get-ChildItem config
```

Должен показаться файл `config.yaml` (или список файлов в `config`).

---

## Шаг 4. Собрать оркестратор (один раз или после изменений кода)

В том же окне PowerShell, будучи в **корне проекта**, выполните **по очереди** две команды (каждая заканчивается Enter):

```powershell
Set-Location orchestrator-java
mvn package
```

Подождите, пока в конце не будет `BUILD SUCCESS`. Если написано `BUILD FAILURE` — прокрутите выше красным, скопируйте текст ошибки и покажите тому, кто разбирается в Java.

Вернитесь в корень проекта:

```powershell
Set-Location ..
```

**Где лежит готовый файл:** `orchestrator-java\target\orchestrator-0.1.0-SNAPSHOT.jar`  
Если этого файла нет — сборка не удалась.

---

## Шаг 5. Собрать сервис геометрии (нужно оркестратору по конфигу)

Снова из **корня проекта**:

```powershell
Set-Location java-geometry-service
mvn package
Set-Location ..
```

Должен появиться файл `java-geometry-service\target\java-geometry-service-0.1.0-SNAPSHOT.jar`.  
В `config\config.yaml` оркестратор как раз вызывает этот jar по относительному пути — поэтому запускать оркестратор нужно из корня проекта (см. следующий шаг).

---

## Шаг 5а. Виртуальное окружение детектора (kopcheniservice)

Оркестратор по умолчанию запускает **не** старый `python-detectors`, а сервис из папки `kopcheniservice` через скрипт `kopcheniservice\run_stdio_worker.py`. Ему нужен Python с зависимостями в **`kopcheniservice\backend\.venv`**.

1. Откройте PowerShell.
2. Выполните (путь к корню проекта подставьте свой):

```powershell
Set-Location "D:\VisionMaster4.4.0\megaOrchestratoraue228-main\kopcheniservice\backend"
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

3. Деактивировать среду не обязательно. Главное — чтобы существовали файлы:
   - `kopcheniservice\backend\.venv\Scripts\python.exe` (Windows),  
   - или `kopcheniservice/backend/.venv/bin/python` (Linux).

Если `python -m venv` выдаёт ошибку — установите Python 3 с сайта python.org и повторите.

---

## Шаг 6. Запустить оркестратор

1. PowerShell должен быть открыт в **корне проекта** (где папка `config`).
2. Выполните:

```powershell
java -jar orchestrator-java\target\orchestrator-0.1.0-SNAPSHOT.jar
```

Если оркестратор ругается на отсутствие конфига или **не находит** `kopcheniservice\backend\.venv\...`, хотя файлы на месте — вы, скорее всего, запускаете Java **не из корня проекта** (например, двойной щелчок по jar из другой папки). Тогда укажите конфиг **полным путём**:

```powershell
java -jar orchestrator-java\target\orchestrator-0.1.0-SNAPSHOT.jar "D:\полный\путь\к\репозиторию\config\config.yaml"
```

Корень проекта для дочерних процессов вычисляется из расположения YAML (два уровня вверх от файла), поэтому такой запуск обычно решает проблему.

**Что должно произойти:** окно не закрывается сразу, в нём бегут строки логов. Это нормально — программа работает.

**Если окно сразу закрылось** — так не должно быть. Запустите снова, но из папки `orchestrator-java`:

```powershell
Set-Location orchestrator-java
java -jar target\orchestrator-0.1.0-SNAPSHOT.jar ..\config\config.yaml
```

Так вы явно укажете путь к конфигу. Если снова вылет — в том же окне после команды будет текст ошибки: его нужно прочитать.

**Если написано, что не найден `config\config.yaml`** — вы не в корне проекта. Выполните `Set-Location` на ваш корень (шаг 3) и повторите команду из шага 6.

**Если в логах пишут про `camera-worker binary not found`** — оркестратор **запустился**, но основной цикл с камерами **не стартует**, пока не собран отдельный исполняемый файл `camera-worker`. Это уже не «запуск jar», а сборка C++ проекта в папке `camera-worker`. Для первого знакомства с оркестратором попросите разработчика собрать воркер или временно не обращайте внимания на отсутствие интеграции.

---

## Шаг 7. Остановить оркестратор

В окне PowerShell, где он запущен, нажмите **Ctrl+C**. Подтвердите остановку, если спросят.

---

## Если нужен другой конфиг (не обязательно)

По умолчанию берётся файл `config\config.yaml` относительно **текущей папки** в PowerShell.

Запуск с явным путём к YAML (подставьте свой путь):

```powershell
java -jar orchestrator-java\target\orchestrator-0.1.0-SNAPSHOT.jar "D:\полный\путь\к\моему.yaml"
```

---

## Коды выхода (если программа сама вышла с цифрой)

- **1** — не найден файл конфигурации.
- **2** — ошибка при старте (например, битый YAML).

## Сервис геометрии отдельно (обычно не нужно руками)

Оркестратор сам поднимает geometry по настройкам в `config.yaml`.  
Запуск jar вручную **без оркестратора** почти бесполезен: программа ждёт специальные бинарные команды в stdin, это для разработчиков.

Если очень нужно проверить, что jar собрался:

```powershell
java -jar java-geometry-service\target\java-geometry-service-0.1.0-SNAPSHOT.jar
```

Окно «зависнет» или сразу выйдет — так и задумано без внешнего клиента.

## Swagger (картинка API для фронтендера)

Нужен **интернет в браузере** (страница тянет скрипты с сайта unpkg).

- Если в `config.yaml` включён блок `ui_http` (`enabled: true`), откройте в браузере:  
  `http://127.0.0.1:8099/swagger`  
  (цифра порта — из `ui_http.port`, чаще всего 8099.)

- Заглушка fan-out почти всегда поднимается вместе с интеграцией:  
  `http://127.0.0.1:8088/swagger`  
  (порт из `fanout.client_http.port`, в примере конфига — 8088.)

На странице Swagger вверху выберите **Server** с тем же портом, с которого вы открыли страницу, иначе кнопка «Execute» будет стучаться не туда.

---

## Короткая шпаргалка (когда всё уже установлено)

Из корня проекта в PowerShell:

```powershell
Set-Location orchestrator-java; mvn package; Set-Location ..
Set-Location java-geometry-service; mvn package; Set-Location ..
java -jar orchestrator-java\target\orchestrator-0.1.0-SNAPSHOT.jar
```

Если что-то из этого не работает — сохраните **полный текст** из окна PowerShell с момента команды до конца и передайте его тому, кто помогает с настройкой.
