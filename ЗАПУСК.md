# Kal — инструкция для запуска (скопировать в другой чат)

> **Задача для ассистента:** по этому файлу подготовь мне **готовые команды под мою ОС** (Linux или Windows), которые я вставлю в терминал **по порядку**.  
> Подставь абсолютный путь к репозиторию: `REPO_ROOT=/home/oleg/Загрузки/realise/Kal` (или спроси у пользователя).  
> Не пропускай шаги сборки, если бинарников/jar ещё нет.

---

## 0. Контекст проекта

**Kal** — система визуальной инспекции. Главный процесс — Java-оркестратор.

Цепочка одного цикла:

`вспышка (LightServer) → camera-worker (захват в SHM) → java-geometry → analisSurface (Python HTTP) → решение → fan-out`

| Компонент | Путь в репо | Как запускается |
|-----------|-------------|-----------------|
| Оркестратор | `orchestrator-java/` | **вручную** `java -jar ...` |
| Детектор Python | `analisSurface/backend/` | **вручную** uvicorn :8000 |
| Геометрия | `java-geometry-service/` | **оркестратор** поднимает jar |
| Камера | `camera-worker/` | **оркестратор** поднимает бинарник |
| Подсветка COM | `LightServer/` | **оркестратор** dotnet :5079 |
| Подсветка MV-LE | `LightServerv.v2/` | **оркестратор** dotnet :5080 |
| Fan-out заглушка | — | **оркестратор** HTTP :8088 |

Конфиг: `config/config.yaml` (оркестратор) + `config/config.json` (camera-worker, поля должны совпадать).

**Режим сейчас:** `integration.dev_auto_trigger_stub.enabled: true` — инспекция **каждые 5 с** без внешнего триггера (временная заглушка).

## 1. Требования к машине

| Инструмент | Версия |
|------------|--------|
| JDK | **17+** |
| Maven | 3.8+ |
| Python | 3.10+ |
| .NET SDK | **10** (`net10.0` в LightServer) |
| CMake | 3.16+ (для camera-worker) |

**Linux + камера Aravis:** `pkg-config`, `libarv-0.8-dev`, `python3-dev`, `python3-numpy`  
**Windows + камера Hik:** MVS SDK + сборка camera-worker под Windows  
**Без камеры (отладка):** `capture_source: fake` в **обоих** `config.yaml` и `config.json`

Проверки (ассистент должен включить в подготовку):

```bash
java -version
mvn -version
python3 --version
dotnet --version
```

## 3. Сборка (один раз или после изменений кода)

### 3.1. Java: оркестратор

```bash
cd "$REPO_ROOT/orchestrator-java"
mvn package -DskipTests
cd "$REPO_ROOT"
# артефакт: orchestrator-java/target/orchestrator-0.1.0-SNAPSHOT.jar
```

### 3.2. Java: геометрия

```bash
cd "$REPO_ROOT/java-geometry-service"
mvn package -DskipTests
cd "$REPO_ROOT"
# артефакт: java-geometry-service/target/java-geometry-service-0.1.0-SNAPSHOT.jar
```

### 3.3. camera-worker

**Перед сборкой:** в `config/config.yaml` и `config/config.json` одинаковый `capture_source`:

- `fake` — без камеры (рекомендуется для первого запуска)
- `aravis` — Linux GigE
- `hik` — Windows + MVS SDK

**Linux (fake или aravis):**

```bash
# fake — минимум пакетов:
# sudo apt install -y cmake python3-dev python3-numpy

# aravis — дополнительно:
# sudo apt install -y pkg-config libarv-0.8-dev

cd "$REPO_ROOT/camera-worker"
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
cd "$REPO_ROOT"
# артефакт: camera-worker/build/camera_worker
```

**Windows:** сборка через CMake/VS с MVS SDK — см. `camera-worker/CMakeLists.txt`, артефакт `camera-worker/build/Release/camera_worker.exe` или `camera_worker.exe`.

### 3.4. LightServer (.NET)

```bash
dotnet build "$REPO_ROOT/LightServer/LightServer.csproj" -c Release
dotnet build "$REPO_ROOT/LightServerv.v2/LightServer.csproj" -c Debug
```

Ожидаемые DLL (пути из конфига):

- `LightServer/bin/Release/net10.0/LightServer.dll` → http://127.0.0.1:5079
- `LightServerv.v2/bin/Debug/net10.0/LightServer.dll` → http://127.0.0.1:5080

Без железа подсветки LightServer уходит в симуляцию (`SimulateIfInitFailed` в appsettings).

### 3.5. Python: venv analisSurface (один раз)

```bash
cd "$REPO_ROOT/analisSurface/backend"
python3 -m venv .venv
source .venv/bin/activate    # Windows: .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
deactivate
cd "$REPO_ROOT"
```

## 4. Запуск (каждый раз) — ДВА терминала

### Терминал 1 — analisSurface (обязательно, первым)

Оркестратор **не стартует**, если :8000 недоступен.

```bash
cd "$REPO_ROOT/analisSurface/backend"
source .venv/bin/activate
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Проверка в другом окне:

```bash
curl -s http://127.0.0.1:8000/detector/health
```

Должен быть JSON со статусом ok. URL = `python_detector.base_url` в `config/config.yaml`.

---

### Терминал 2 — оркестратор

```bash
cd "$REPO_ROOT"
java -jar orchestrator-java/target/orchestrator-0.1.0-SNAPSHOT.jar "$CONFIG"
```

Или без `$CONFIG`, если уже в корне репо:

```bash
cd "$REPO_ROOT"
java -jar orchestrator-java/target/orchestrator-0.1.0-SNAPSHOT.jar
```

**Остановка:** Ctrl+C в обоих терминалах (сначала оркестратор, потом uvicorn).

## 5. Что должно быть в логах при успехе

- `python detector transport=http base_url=http://127.0.0.1:8000`
- `java-geometry health => ...`
- `worker cam=0 health type=...`
- `dev_auto_trigger_stub enabled interval_ms=5000`
- каждые ~5 с — новый цикл capture / inspect / fanout

JPEG кадров (если включено): каталог `testimage/` в корне репо (`integration.save_captures`).

## 7. Важные флаги в config/config.yaml

```yaml
capture_source: "fake"   # для первого запуска без камеры — то же в config.json

integration:
  light_server_startup_delay_ms: 5000
  dev_auto_trigger_stub:
    enabled: true
    interval_ms: 5000
  continuous_inspection:
    enabled: false

python_detector:
  base_url: "http://127.0.0.1:8000"

light_servers:
  enabled: true
  fail_on_error: false   # пайплайн не падает, если свет недоступен
```

Когда появится триггер с линии: `dev_auto_trigger_stub.enabled: false`.

## 8. Типичные ошибки

| Симптом | Причина | Действие |
|---------|---------|----------|
| `analisSurface FastAPI pool is empty` | uvicorn не запущен | Терминал 1, шаг 4 |
| `camera-worker binary not found` | не собран worker | Шаг 3.3 |
| capture failed / hik on Linux | неверный `capture_source` | поставить `fake` в yaml **и** json |
| LightServer failed to start | нет .NET 10 / нет DLL | шаг 3.4 или отключить endpoint в yaml |
| оркестратор сразу вышел | нет Python на :8000 | лог ERROR выше |
| exit code 1 | нет config.yaml | запуск из `REPO_ROOT` или передать путь к yaml |
| exit code 2 | битый YAML | проверить синтаксис |
